# 逗子ライブインクルーシブ Webサイトリプレース 要件定義書

## 1. プロジェクト概要

### 1.1 プロジェクト名
逗子ライブインクルーシブ 公式Webサイト リプレース

### 1.2 背景・目的
障がい者アーティストとトップアーティストが共演するインクルーシブなコンサート「逗子ライブインクルーシブ」の公式サイトを、現代的な技術スタックで再構築する。

### 1.3 リプレースの目的
- **デザイン完全刷新**: 新しいビジュアルデザインで、より洗練された体験を提供
- **コード品質改善**: 保守性・拡張性の高いコードベースへ
- **パフォーマンス改善**: 表示速度・SEOの最適化

## 3. 機能要件

### 3.1 ページ構成

| ページ | URL | 概要 |
|--------|-----|------|
| トップページ | `/` | イベント概要、日程、ハイライト |
| 出演者 | `/performers` | 出演アーティスト一覧・詳細 |
| チケット | `/tickets` | チケット情報・購入導線 |
| 会場 | `/venue` | 会場情報・アクセス・バリアフリー情報 |
| 過去公演 | `/archive` | 過去の公演アーカイブ |
| About | `/about` | コンセプト・理念 |
| お問い合わせ | `/contact` | お問い合わせフォーム |

### 3.2 機能一覧

#### 3.2.1 コンテンツ表示
- [ ] イベント基本情報の表示（日時、会場、料金）
- [ ] 出演者情報の動的表示（CMSから取得）
- [ ] チケット情報・購入リンク
- [ ] 会場情報・Googleマップ埋め込み
- [ ] 過去公演のギャラリー・動画埋め込み
- [ ] SNSリンク（Facebook, Instagram, X）

#### 3.2.2 インタラクション
- [ ] お問い合わせフォーム
- [ ] レスポンシブナビゲーション
- [ ] スムーズスクロール

#### 3.2.3 外部連携
- [ ] クラウドファンディング（CAMPFIRE）リンク
- [ ] SNSシェア機能
- [ ] Google Analytics（任意）

### 3.3 コンテンツ管理
以下のコンテンツをCMSで管理:
- 出演者情報（名前、プロフィール、写真、SNS）
- イベント情報（日時、会場、料金）
- お知らせ・ニュース
- 過去公演アーカイブ

## 4. 非機能要件

### 4.1 パフォーマンス
| 指標 | 目標値 |
|------|--------|
| Lighthouse Performance | 90以上 |
| First Contentful Paint | 1.5秒以下 |
| Largest Contentful Paint | 2.5秒以下 |
| Cumulative Layout Shift | 0.1以下 |

### 4.2 アクセシビリティ（WCAG 2.1 AA準拠）
- [ ] 適切なセマンティックHTML
- [ ] キーボードナビゲーション対応
- [ ] スクリーンリーダー対応（ARIA属性）
- [ ] 十分なコントラスト比（4.5:1以上）
- [ ] フォーカス表示の明確化
- [ ] 画像への代替テキスト
- [ ] フォームのラベル・エラー表示
- [ ] 動画への字幕対応（該当する場合）

### 4.3 SEO
- [ ] 適切なメタタグ（title, description, OGP）
- [ ] 構造化データ（JSON-LD）
- [ ] sitemap.xml
- [ ] robots.txt
- [ ] canonical URL

### 4.4 セキュリティ
- [ ] HTTPS必須
- [ ] 環境変数によるシークレット管理
- [ ] フォーム入力のサニタイズ
- [ ] CSP（Content Security Policy）設定
